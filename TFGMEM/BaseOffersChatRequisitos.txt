Tabla offers:
La tabla offers, mostrada en la tabla 4.9 cuenta con un campo id que actúa como clave primaria. Las ofertas pueden llamarse igual o pedir las mismas características por ello, para distinguir una de otras, se les asocia el campo id nada más se crean. 
El campo owener_id actúa como clave foránea ya que está relacionado con el campo id de la tabla users, sin embargo, lo importante de este campo es que el id asociado debe tener en la tabla users el rol con valor de company, ya que solo estas pueden subir ofertas.
El campo vacants representa las plazas libres que tiene la empresa para dicha oferta de trabajo.
El campo document_path almacena la ruta del fichero que quiere subir la compañía (las ofertas cuentan con un archivo de texto que se puede descargar). La compañía es la que debe subir dicho archivo y en este campo se guarda la ruta donde se almacenará dicho archivo en el servidor (dicha ruta se define en el código).
Tabla conversations:
La tabla conversations, representada en la tabla 4.10 tiene un campo id como clave primaria. Cada conversación entre dos usuarios es única y en el momento en el que cualquiera de los dos envía el primer mensaje, se crea una conversación y a esta se le asocia dicho id.
Tabla messages:
En la tabla messages mostrada en la tabla 4.11 el capo id actúa como clave primaria. Un mismo mensaje puede ser enviado en numerosas conversaciones por lo que cada mensaje en el momento en el que alguien lo envía se almacena y se le asigna un id para poder identificarlo.
El mensaje va asociado a una conversación (id_conversation), ha sido enviado por un usuario (id_sender) y ha sido recibido por otro (id_receiver).

Requisitos funcionales:

Subida de ofertas: Los usaurios que tengan como rol company podrán subir ofertas de empleo indicando lo que buscan el salario y las plazas disponibles.

Búsqueda de ofertas: Los usuarios tendrán herramientas para encontrar ofertas que les interesen.

Descarga de ofertas: Los usuarios podrán descargar la oferta de trabajo subida por las empresas.

Editar ofertas: Los usuarios que tengan como rol company podrán editar una oferta ya subida por ellos para cambiar algo que no les convenza.

Editar vídeo : Los usuarios podrán editar sus videos una vez los hayan subido para cambiar algo que no les convenza.

Enviar y recibir mensajes: Los usuarios podrán iniciar un chat en la oferta que les interese, enviar mensajes a dicha empresa así como recibirlos.

Visualizar char: Los usuario podrán ver los mensajes enviados y recibidos con aquellas empresas que hayan hablado previamente.
